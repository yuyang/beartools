# Audit 清单（按需使用）

默认不单独执行 audit。仅在涉及安全、持久化数据、外部发布、敏感配置，或用户明确要求时使用。

audit 阶段只读代码，不修改文件。目标是判断改动是否适合进入商业开发的合并或发布链路。

## 风险维度
- 安全：密钥、令牌、用户数据、命令执行、路径访问、外部请求。
- 数据：数据库结构、数据迁移、兼容性、幂等性、回滚。
- 运行：日志、配置、超时、重试、并发、资源释放。
- 交付：测试门禁、文档、示例配置、发布说明。
- 协作：是否需要产品、架构、运维或安全确认。

## 必查项
- 是否把敏感信息写入仓库、日志或测试快照。
- 是否新增网络访问、文件写入、shell 调用或权限扩大。
- 是否涉及数据库表结构或持久化格式变化。
- 是否有清晰失败信息，方便排查。
- 是否保持默认不向 console 打日志。
- 是否需要更新 `config/beartools.yaml.sample` 或用户文档。
- 是否需要人工确认业务规则、状态流转或兼容策略。
- 是否有可执行的回滚或降级路径。

## 输出格式
```text
Audit Result: pass / conditional-pass / block

Blocking Risks
- ...

Conditional Risks
- ...

Release Notes
- ...

Human Decisions
- ...
```
