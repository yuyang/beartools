# Review 清单

review 阶段只读代码，不修改文件。目标是尽早发现会影响合并质量的问题。

## 优先级
1. 行为 bug、回归风险、数据损坏风险。
2. 测试缺口，尤其是新增行为、错误路径、边界条件。
3. 与 `AGENTS.md` 冲突的实现方式。
4. 过度设计、重复逻辑、难以维护的抽象。
5. 性能问题和资源泄漏。

## 必查项
- 当前 diff 是否完整满足需求。
- 是否引入未说明的外部行为变化。
- 是否误改无关模块或扩大范围。
- 错误处理是否捕获了具体异常类型。
- 是否存在空 `except` 或直接吞异常。
- 所有新增/修改函数是否有参数和返回值类型提示。
- 新增依赖是否在 `pyproject.toml` 中使用 `==` 固定版本。
- 测试是否覆盖成功路径、失败路径和关键边界。
- CLI 输出是否被日志或调试信息污染。

## 输出格式
有问题时：
```text
Findings
- [P1] 文件:行号 问题说明

Open Questions
- 需要用户确认的问题

Test Gaps
- 未覆盖或无法验证的部分
```

没有问题时：
```text
未发现阻塞性问题。
剩余风险：...
测试覆盖：...
```
